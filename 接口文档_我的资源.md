# 接口一：查询我的所有资源

---

## 一、接口说明

### 接口路径
```
POST /dsjpt/jk/resource/my/list.xhtml
```

### 接口功能
查询当前登录用户的所有资源，包括：
1. **我创建的资源**（Skill、MCP、数字警员）
2. **授权给我的资源**（Skill、MCP、数字警员）

### 权限要求
- 已登录用户（需要 Token）
- 无特殊角色要求

### 特性
- ✅ 自动合并创建和授权的资源
- ✅ 自动去重（避免同一资源重复出现）
- ✅ 支持按资源类型筛选
- ✅ 统一返回格式
- ✅ 包含详细统计信息

---

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | String | 是 | 用户登录令牌 |

### 请求体（JSON）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resource_type | String | 否 | 资源类型筛选：<br>`skill` - 只查询 Skill<br>`mcp` - 只查询 MCP<br>`officer` - 只查询数字警员<br>不传或为空则查询所有类型 |

### 字段说明

#### data.list
| 字段 | 类型 | 说明 |
|------|------|------|
| skills | Array | Skill 资源列表 |
| mcps | Array | MCP 服务列表 |
| officers | Array | 数字警员列表 |

#### data.statistics
| 字段 | 类型 | 说明 |
|------|------|------|
| total | Integer | 资源总数 |
| skills | Integer | Skill 总数 |
| mcps | Integer | MCP 总数 |
| officers | Integer | 数字警员总数 |
| created | Object | 我创建的资源统计 |
| created.skills | Integer | 我创建的 Skill 数量 |
| created.mcps | Integer | 我创建的 MCP 数量 |
| created.officers | Integer | 我创建的数字警员数量 |
| authorized | Object | 授权给我的资源统计 |
| authorized.skills | Integer | 授权给我的 Skill 数量 |
| authorized.mcps | Integer | 授权给我的 MCP 数量 |
| authorized.officers | Integer | 授权给我的数字警员数量 |

#### 资源对象通用字段
| 字段 | 类型 | 说明 |
|------|------|------|
| _source | String | 资源来源：<br>`created` - 我创建的<br>`authorized` - 授权给我的 |
| _auth_info | Object | 授权信息（仅授权资源有） |

#### _auth_info 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| auth_id | Integer | 授权记录 ID |
| auth_type | String | 授权类型：<br>`read` - 只读权限<br>`write` - 可写权限 |
| expire_time | String | 授权过期时间 |
| auth_time | String | 授权时间 |

## 五、使用说明

### 1. 资源来源标识（_source）

| 值 | 说明 |
|------|------|
| `created` | 我创建的资源 |
| `authorized` | 授权给我的资源 |

### 2. 授权信息（_auth_info）

仅当 `_source: "authorized"` 时存在此字段。

| 场景 | 说明 |
|------|------|
| 有 `_auth_info` | 资源是授权给我的 |
| 无 `_auth_info` | 资源是我创建的 |

### 3. 权限类型（auth_type）

| 值 | 说明 |
|------|------|
| `read` | 只读权限（可以使用资源） |
| `write` | 可写权限（可以修改资源配置） |

### 4. 授权过期时间

- `expire_time` 为 `null` 表示永久有效
- 过期时间格式：`YYYY-MM-DD HH:mm:ss`

## 六、前端集成示例

### JavaScript / TypeScript
```javascript
async function getMyResources(resourceType = null) {
  const response = await fetch('/dsjpt/jk/resource/my/list.xhtml', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'token': localStorage.getItem('token')
    },
    body: JSON.stringify({
      resource_type: resourceType // 'skill' | 'mcp' | 'officer' | null
    })
  });
  
  const result = await response.json();
  
  if (result.status === 'succeed') {
    return result.data;
  } else {
    throw new Error(result.message);
  }
}

// 使用示例
getMyResources().then(data => {
  console.log('所有资源:', data.list);
  console.log('统计信息:', data.statistics);

    ...data.list.mcps.filter(m => m._source === 'authorized'),

  ];
});
```

## 七、常见问题

### Q1: 为什么同一个资源既出现在 created 又出现在 authorized？
**A**: 接口已自动去重，不会出现这种情况。如果资源是自己创建的，即使后来也授权给了自己，也只会出现在 `created` 列表中。

### Q2: 如何判断资源是否即将过期？
**A**: 检查 `_auth_info.expire_time` 字段，与当前时间比较即可。
```javascript
const expireTime = new Date(item._auth_info.expire_time);
const now = new Date();
const daysLeft = Math.floor((expireTime - now) / (1000 * 60 * 60 * 24));
if (daysLeft < 7) {
  console.warn('授权即将过期，剩余天数:', daysLeft);
}
```

### Q3: 如何区分资源的权限类型？
**A**: 查看 `_auth_info.auth_type` 字段：
- `read` - 只读权限
- `write` - 可写权限

### Q4: 为什么授权给我的资源没有 _auth_info？
**A**: 正常情况下授权资源都会有 `_auth_info`。如果没有，可能是数据异常，建议检查数据库授权记录。

# 接口二：查询所有公开/上架资源

**接口版本**：v1.0  
**最后更新**：2026-06-29

---

## 一、接口说明

### 接口路径
```
POST /dsjpt/jk/auth/resources.xhtml
```

### 接口功能
查询所有可授权的资源（支持公开/上架资源筛选），包括：
1. **Skill 资源**
2. **MCP 服务**
3. **数字警员**

### 权限要求
- 已登录用户（需要 Token）
- 管理员角色优先（查看可授权资源）

### 特性
- ✅ 支持三种资源类型统一查询
- ✅ 支持按公开状态筛选（is_public）
- ✅ 支持按部门 ID 筛选
- ✅ 支持按创建者 ID 筛选
- ✅ 支持关键词搜索
- ✅ 支持分页

---

## 二、请求参数

### 请求头
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| token | String | 是 | 用户登录令牌 |
| Content-Type | String | 是 | 固定为 `application/json` |

### 请求体（JSON）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resource_type | String | 否 | 资源类型筛选：<br>`skill` - 只查询 Skill<br>`mcp` - 只查询 MCP<br>`officer` - 只查询数字警员<br>不传则查询所有类型 |
| is_public | Boolean | 否 | 是否公开/上架：<br>`true` - 公开/上架资源<br>`false` - 未公开资源<br>不传则查询所有 |

| page | Integer | 否 | 页码（默认 1） |
| limit | Integer | 否 | 每页数量（默认 20） |

## 六、使用说明

### 1. 公开状态（is_public）

| 值 | 说明 |
|------|------|
| `true` | 公开/上架资源（已通过两级审核） |
| `false` | 未公开资源（审核中或未提交审核） |

### 2. 审核状态码

| 状态码 | 含义 | 说明 |
|--------|------|------|
| `00` | 静默/待审核 | 创建后默认状态 |
| `01` | 申请中 | 已提交申请，等待审核 |
| `02` | 审核通过 | 审核已通过 |
| `03` | 已拒绝 | 审核被拒绝 |

### 3. 资源类型

| 资源类型 | 说明 |
|---------|------|
| `skill` | 技能 |
| `mcp` | MCP 服务 |
| `officer` | 数字警员 |

### 4. 典型使用场景

#### 场景 1：资源授权页面
```javascript
// 查询所有公开资源，供管理员选择授权
const resources = await getAuthResources({ is_public: true });
```

#### 场景 2：按部门查看资源
```javascript
// 查看技术部的所有公开资源
const resources = await getAuthResources({
  dept_id: 5,
  is_public: true
});
```

#### 场景 3：查看某人创建的资源
```javascript
// 查看张三创建的所有资源
const resources = await getAuthResources({
  creator_id: 37
});
```

## 七、前端集成示例

### JavaScript / TypeScript
```javascript
async function getAuthResources(params = {}) {
  const response = await fetch('/dsjpt/jk/auth/resources.xhtml', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'token': localStorage.getItem('token')
    },
    body: JSON.stringify(params)
  });
  
  const result = await response.json();
  
  if (result.status === 'succeed') {
    return result.data;
  } else {
    throw new Error(result.message);
  }
}

// 使用示例
getAuthResources({ is_public: true, resource_type: 'skill' })
  .then(data => {
    console.log('公开 Skill 资源:', data.list);
    console.log('总数:', data.total);
  });
```

## 八、常见问题

### Q1: 如何只查询已上架的资源？
**A**: 设置 `is_public: true` 参数即可。

### Q2: 为什么查询结果包含未审核通过的资源？
**A**: 如果不传 `is_public` 参数，会返回所有资源（包括审核中和已拒绝的）。建议设置 `is_public: true` 只查询已上架资源。

### Q3: 如何查看某个部门的所有资源？
**A**: 使用 `dept_id` 参数，例如：`{ dept_id: 5 }`

### Q4: 支持模糊搜索吗？
**A**: 支持，使用 `keyword` 参数可以对资源名称和编码进行模糊搜索。
